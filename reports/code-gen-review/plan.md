# Research Plan: Effective Methods for Reviewing Coding Agent Output

## Main Research Question
What are the most effective methods, tools, and practices for reviewing code produced by AI Coding Agents (e.g., Cursor, Copilot, Claude Code, etc.)?

## Subtopic 1: Automated Review Tools & Techniques for AI-Generated Code
Investigate tools and automated processes specifically designed or adapted for reviewing AI-generated code. Includes linters, static analysis, AI-powered code review tools, and CI/CD integration patterns.

## Subtopic 2: Human Review Strategies & Checklists for LLM Code
Explore best practices, checklists, and structured human review approaches tailored to code produced by LLMs. Includes what to look for beyond traditional code review.

## Subtopic 3: Common Failure Modes & Quality Issues in AI-Generated Code
Identify the most frequent bugs, security vulnerabilities, logic errors, and anti-patterns that appear in LLM-generated code, so reviewers know what to prioritize.

## Synthesis Approach
Cross-reference findings to produce a practical guide covering: (1) automated checks to run first, (2) human review strategies with concrete checklists, and (3) priority areas to inspect based on known failure modes.
