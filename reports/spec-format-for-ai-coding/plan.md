# Research Plan: Spec/Test Input Formats for AI Coding Agents

## Main Research Question
What are the most suitable Spec/Test input formats for Coding Agents to stably implement software?

## Subtopics

### 1. Spec Formats Used for AI Coding Agents
Investigate known spec formats used in AI-assisted development:
- Gherkin/Cucumber (Given-When-Then)
- Markdown-based specs (e.g., SpecStory, conventional commit specs)
- TDD (Test-Driven Development) with unit tests as specs
- Docstring/signature-based specs
- Formal specification languages (TLA+, Alloy)
- How Cursor, GitHub Copilot, Devin, and other AI coding tools handle specs

### 2. Best Practices for Writing Specs that AI Can Implement
What makes a spec effective for AI implementation?
- Level of detail and granularity
- Unambiguous language and structure
- Including edge cases and error handling
- Relationship between spec and test cases
- Examples from successful AI coding workflows

### 3. Comparative Analysis of Spec-First Approaches
Compare different approaches:
- Spec-first vs Test-first vs Prompt-first
- Iterative refinement of specs
- How specs reduce hallucination and improve reliability
- Tradeoffs between formality and flexibility

## Synthesis
Combine findings into actionable recommendations for choosing/creating spec formats.
