# 異常體調查與研究遊戲設計（Anomaly Research Game Design Document）

玩家扮演異常現象研究機構主管，負責研究與收容未知異常體。核心玩法不是戰鬥，而是透過有限實驗推理異常規則：觀察 → 假說 → 實驗 → 驗證 → 建檔。

---

# 核心遊戲循環

取得異常體 → 閱讀初始情報 → 設計實驗 → 執行調查（消耗資源） → 獲得觀測結果 → 提出假說 → 驗證假說 → 逐步補完研究檔案 → 完成結案報告 → 獲得獎勵 → 下一異常體

---

# 勝利條件

完成異常體研究檔案：名稱、危險等級、外觀描述、觸發條件、影響對象、異常效果、收容方式、特殊規則

---

# 失敗條件

研究資源耗盡、研究員全滅或精神崩潰、異常體突破收容、超時未完成研究

---

# 異常體核心模型（DSL）

所有異常體由統一規則構成：

Trigger → 觸發條件
Condition → 判定條件
Target → 影響目標
Effect → 異常效果

範例1：Trigger=接觸，Condition=目標受傷，Target=接觸者，Effect=即死
範例2：Trigger=被注視，Condition=超過10秒，Target=最近的人，Effect=精神崩潰

---

# 調查系統（有限通用操作）

玩家只有少量通用工具，但可覆蓋所有異常體。

觀察 Observe：低成本無風險，獲得外觀變化、行為模式、環境反應，用於建立初步假說。
接觸 Interaction：派遣研究員靠近異常體，可能無反應、逃離、攻擊、模仿或污染，用於測試互動規則。
刺激 Stimulus：施加光、聲音、熱、冷、電、震動，用於測試觸發條件與反應弱點。
物件投入 Object Test：投入帶Tag物品（金屬、有機、液體、電子、武器、鏡面、活體），用於測試偏好或排斥。
對象測試 Subject Test：使用不同研究員（勇氣、理智、觀察力、溝通、體力與受傷/健康/性別/年齡標籤）測試目標條件。
隔離測試 Isolation：切換環境（有光、黑暗、有/無人、封閉、開放、真空）測試環境觸發規則。
長期觀察：觀察週期行為、移動規律與成長變化。
解除收容 Breach Test：主動破壞收容觀察追蹤、攻擊與移動規則，高風險高回報。

---

# 資訊輸出系統

遊戲只提供觀測結果，不提供結論。例如：實驗「強光照射」結果「異常體後退3公尺」，玩家自行推理假說，但真相可能不同（例如實際是對高溫敏感）。

---

# 假說系統

玩家提出模型並透過實驗修正可信度：例如「異常體攻擊接觸者」可能從35%驗證成功上升到80%，或驗證失敗下降到5%。

# 研究檔案系統（進度制）

效果 █████░░░░░ 50%
收容方式 ██░░░░░░░░ 20%

---

# 研究員系統

屬性：勇氣、理智、觀察力、溝通、體力
狀態：健康、受傷、重傷、瘋狂、死亡、失蹤

低能力研究員會產生模糊報告如「好像動了一下」，高能力研究員產生精確報告如「13:24發生17cm位移」。

---

# 資源系統

研究點數、時間、研究員、設備。設備包括攝影機、熱感儀、無人機、感測器。

---

# 評分系統

完成度、正確率、人員損失、資源效率。

---

# Rogue-lite 成長系統

完成異常體後解鎖新設備、新研究員、新實驗方法與更高危險等級異常體，例如高功率電擊、自動觀測系統、遠端機器人、基因分析。

---

# MVP（最小可行版本）

行動：觀察、接觸、刺激、投入物品、更換研究員、解除收容。
異常模型：Trigger → Condition → Target → Effect。
檔案：名稱、觸發條件、效果、收容方式。

---

# 核心設計理念

透過極少通用實驗工具，讓玩家對最大未知系統進行推理建模，本質是從觀測數據建立假說、設計實驗、驗證或推翻並逐步逼近真相。
